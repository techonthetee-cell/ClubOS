1. **Create Auth.js configuration (apps/dashboard/lib/auth.ts):**
   - Install: `pnpm add next-auth@beta --filter @clubos/dashboard`
   - Configure Auth.js v5 with Credentials provider
   - The authorize function:
     a. Reads `x-tenant-db` header from the request (set by middleware)
     b. Creates ClubOSClient with ODOO_URL env var + tenant db
     c. Calls client.authenticate(email, password)
     d. On success: fetches user details via searchRead on res.users (fields: name, email, groups_id)
     e. Returns user object: { id: uid, name, email, tenantId, tenantDb }
     f. On failure: returns null (Auth.js shows error)
   - Session strategy: JWT
   - JWT callback: includes tenantId and tenantDb in the token
   - Session callback: exposes tenantId and tenantDb in session.user
   - Export: handlers, auth, signIn, signOut

   IMPORTANT: If Auth.js v5 has compatibility issues with Next.js 16 (per Open Question 3 in research), fall back to a simpler JWT approach:
   - Create /api/auth/login route that authenticates with Odoo, creates a JWT, sets httpOnly cookie
   - Create middleware that validates the JWT
   - Document the fallback in the SUMMARY

2. **Create Auth.js route handler:**
