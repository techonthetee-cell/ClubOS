1. **Create packages/ui/ package:**
   - `packages/ui/package.json`: name "@clubos/ui", main "src/index.ts" (no build step -- consumed as source via transpilePackages)
   - Dependencies: react, react-dom (peerDependencies), tailwindcss, class-variance-authority, clsx, tailwind-merge, lucide-react
   - DevDependencies: @types/react, typescript

2. **Initialize shadcn/ui in packages/ui/:**
   - Run `pnpm dlx shadcn@latest init` from packages/ui/ directory
   - Choose: TypeScript, Tailwind CSS v4, src/components path, src/lib/utils.ts path
   - If shadcn CLI has issues with Tailwind v4 in monorepo (Pitfall 6), manually create the component files using shadcn/ui source code as reference

3. **Create emerald design system tokens (packages/ui/src/styles/globals.css):**
   - Use Tailwind v4 CSS-first config with @theme directive
   - Define CSS custom properties for the ClubOS emerald theme:
     - --primary: oklch from emerald-600 (#10B981 / emerald)
     - --primary-foreground: white
     - --secondary: slate-100
     - --secondary-foreground: slate-900
     - --accent: emerald-50
     - --accent-foreground: emerald-900
     - --destructive: red-500
     - --background: white
     - --foreground: slate-950
     - --muted: slate-100
     - --muted-foreground: slate-500
     - --border: slate-200
     - --card: white
     - --card-foreground: slate-950
     - --radius: 0.5rem
   - Dark mode variant (optional, low priority -- use light mode for MVP)

4. **Create utility function (packages/ui/src/lib/utils.ts):**
   - cn() function using clsx + tailwind-merge (standard shadcn pattern)

5. **Add shadcn/ui components (or create manually if CLI fails):**

   a. **Button** (packages/ui/src/components/button.tsx):
      - Variants: default (emerald bg), secondary (slate), outline, ghost, destructive, link
      - Sizes: default, sm, lg, icon
      - Uses class-variance-authority for variant management

   b. **Card** (packages/ui/src/components/card.tsx):
      - Card, CardHeader, CardTitle, CardDescription, CardContent, CardFooter
      - Clean white card with subtle border and shadow

   c. **Input** (packages/ui/src/components/input.tsx):
      - Standard text input with focus ring in emerald
      - Support for label, error state, disabled state

   d. **Badge** (packages/ui/src/components/badge.tsx):
      - Variants: default (emerald), secondary, outline, destructive
      - Used for member status, booking status, etc.

   e. **Avatar** (packages/ui/src/components/avatar.tsx):
      - Image with fallback initials
      - Sizes: sm, md, lg
      - Used for member photos, user avatars in header

   f. **Sidebar** (packages/ui/src/components/sidebar.tsx):
      - Vertical nav sidebar with icon + label items
      - Active state: emerald background highlight
      - Collapsible (icon-only mode)
      - Props: items array of { label, href, icon, active }
      - Slot for logo at top, user info at bottom

   g. **DataTable** (packages/ui/src/components/data-table.tsx):
      - Simple responsive table component
      - Props: columns definition, data array
      - Supports sorting indicator (no client-side sort yet -- that's Phase 2)
      - Used for member lists, tee times, POS orders

6. **Create barrel export (packages/ui/src/index.ts):**
   - Export all components
   - Export cn utility
   - Export globals.css path for consuming apps

IMPORTANT: These components are consumed as SOURCE (not built). The consuming Next.js apps transpile them via `transpilePackages` in next.config.ts. This avoids a separate build step and ensures Tailwind classes are processed correctly.
