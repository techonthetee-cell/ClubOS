1. Create `packages/ai/src/pricing.ts` -- Rule-based dynamic pricing:
   - `PricingFactors { dayOfWeek: number; timeOfDay: string; currentDemand: number; historicalDemand: number; weatherScore: number; isHoliday: boolean; daysUntilTeeTime: number }`
   - `PricingResult { basePrice: number; adjustedPrice: number; multiplier: number; reason: string; factors: { name: string; impact: number; direction: "up" | "down" }[]; confidence: number }`
   - `calculateDynamicPrice(basePrice: number, factors: PricingFactors): PricingResult`
   - Rules:
     - Weekend premium: +20% (Sat), +15% (Sun)
     - Prime time (7-10 AM): +15%
     - Twilight (after 3 PM): -20%
     - High demand (>70% booked): +10-25% scaling
     - Low demand (<30% booked): -10-15%
     - Bad weather (score < 30): -15%
     - Booking within 24h: -10% (last-minute discount)
     - Holiday: +25%
   - Multipliers stack but cap at 0.6x min and 1.5x max
   - Export `generatePricingHeatMap(slots: any[], baseRate: number): Map<string, PricingResult>` for batch processing all slots

2. Create `packages/ai/src/churn.ts` -- Rule-based churn prediction:
   - `ChurnFactors { monthlyVisits: number[]; annualSpend: number; previousAnnualSpend?: number; memberTenure: number; npsScore: number; lastVisitDaysAgo: number; hasOpenComplaints: boolean; missedRenewals: number }`
   - `ChurnResult extends ChurnPrediction { factors: { name: string; weight: number; score: number; direction: "risk" | "safe" }[]; confidencePercent: number }`
   - `predictChurnScore(factors: ChurnFactors): ChurnResult`
   - Scoring rules (weighted):
     - Visit trend (weight 30%): Compare last 3 months avg vs prior 3 months. Decline > 40% = high risk
     - Spending trend (weight 20%): Annual spend vs previous year. Decline > 30% = high risk
     - Recency (weight 20%): Last visit >30 days = medium, >60 days = high
     - NPS (weight 15%): Score <40 = high, 40-60 = medium, >60 = low
     - Tenure (weight 15%): <1 year = slightly higher risk (new members churn more)
   - Map composite score to 1-5 risk scale with thresholds
   - Generate human-readable recommendation string

3. Create `packages/ai/src/insights.ts` -- Member insight generator:
   - `generateInsights(member: { visits: number[]; spend: number; tier: string; risk: number; activities: any[] }): MemberInsight[]`
   - Generate 2-4 insights per member based on patterns:
     - Churn risk > 3: "Visits dropped X% -- recommend personal outreach from GM"
     - High spender + low risk: "Consider upgrade to [next tier] -- spending exceeds tier average by X%"
     - Visit frequency declining but spend steady: "Fewer visits but higher per-visit spend -- may be consolidating visits"
     - Regular diner who hasn't visited grill in 2+ weeks: "Last dining visit was X days ago -- send event invitation"
     - New member (<6 months): "Onboarding check-in due -- schedule welcome call"
     - Top 10% spender: "VIP flag -- ensure staff recognition on next visit"
   - Each insight has type, message, priority, and suggested action

4. Update `packages/ai/src/index.ts`:
   - Replace placeholder functions with real imports from pricing.ts, churn.ts, insights.ts
   - Re-export all types and functions
   - Keep backward compatibility with existing exports
