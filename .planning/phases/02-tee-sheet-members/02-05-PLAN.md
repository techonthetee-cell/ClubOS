  <files>apps/dashboard/app/api/tee-sheet/stream/route.ts, apps/dashboard/app/tee-sheet/hooks/useTeeSheetSSE.ts, apps/dashboard/app/tee-sheet/hooks/useTeeSheet.ts</files>
  <action>
1. Create `apps/dashboard/app/api/tee-sheet/stream/route.ts`:
   - GET handler returning a ReadableStream with SSE headers
   - Headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`
   - In-memory event bus (module-level Map or array) that holds pending events
   - When a booking is created/cancelled/updated, push an event to the bus
   - Stream format: `event: booking-update\ndata: { "type": "created|cancelled|updated", "slot": {...}, "booking": {...} }\n\n`
   - Heartbeat: send `event: heartbeat\ndata: {}\n\n` every 15 seconds to keep connection alive
   - Clean up on disconnect (controller.close or abort signal)
   - Since this is MVP with no Redis, use a simple in-memory pub/sub pattern:
     - Module-level `const listeners = new Set<ReadableStreamDefaultController>()`
     - POST handler on same route (or a separate `/api/tee-sheet/events` route) that pushes events to all listeners
     - Export a `broadcastBookingEvent(event)` function that other API routes can call

   NOTE: Read `node_modules/next/dist/docs/` for Route Handler API changes before implementing.

2. Create `apps/dashboard/app/tee-sheet/hooks/useTeeSheetSSE.ts`:
   - Custom hook: `useTeeSheetSSE(date: string, onEvent: (event) => void)`
   - Creates EventSource connection to `/api/tee-sheet/stream?date={date}`
   - Handles events:
     - `booking-update`: calls `onEvent` with parsed data
     - `heartbeat`: resets reconnect timer
     - `error`: auto-reconnect with exponential backoff (1s, 2s, 4s, max 30s)
   - Shows connection status: `connected | reconnecting | disconnected`
   - Cleanup: closes EventSource on unmount or date change
   - Return: `{ isConnected, reconnectCount, lastEventAt }`

3. Update `useTeeSheet.ts`:
   - Integrate SSE: when a booking is created/cancelled locally, also POST to `/api/tee-sheet/events` to broadcast
   - When SSE event received, merge into local state (update the matching slot)
   - Add a small "Live" indicator (green dot) in the hook's returned state: `isLive: boolean`
   - Add connection status indicator data for the UI

4. Add to tee sheet page.tsx:
   - Small "Live" badge in the top controls bar showing SSE connection status
   - Green dot + "Live" when connected
   - Yellow dot + "Reconnecting..." when reconnecting
   - Red dot + "Offline" when disconnected
   - Toast-style notification when a remote booking update arrives: "J. Harrington booked 9:00 AM Hole 1"
